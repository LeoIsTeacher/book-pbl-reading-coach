# 教练式共读作业反馈 Skill

## 输入与定位

先确认 `book_id`、书名、版本、`task_id`、成员编号，以及成员本次作业。`book_id` 使用小写 ASCII 代码，书名用于展示。再读取项目根目录中的 `books/<book_id>/book.yaml`、`tasks/task-<task_id>.md` 与 `rubrics/task-<task_id>.md`，以及成员现有成长档案。更新档案时读取 `references/growth-profile.md`。

若书籍包、任务或已确认的批改标准缺失，说明缺失项并暂停批改，不凭空生成参考答案，也不借用其他书籍的标准。不同版本的章节范围必须以该书籍包中的版本信息为准。

## 工作流程

1. 解析书籍包，核对书名、版本与任务编号，避免串书、串任务或串档。
2. 检查作业是否回应当前任务要求和指定阅读范围。
3. 依据当前任务的批改指南判断关键变量、书本观点使用、证据、推理、现实约束和高价值思考。
4. 生成给成员的快速反馈，控制在200至600字。
5. 暂不修改正式成长档案，只保留档案更新草稿。
6. 等待主理人明确回复“已发送给成员”。
7. 确认后输出完整最新版成长档案、变化摘要和下一次重点观察项。

## 快速反馈格式

只保留三个动作：

**最有价值的思考**：指出成员答案中最值得保留、最具个人证据或最接近专家思维的一处。

**当前遗漏的关键变量**：只选择一个最影响结论的缺口，不把所有问题一次性列出。

**深入思考的方向**：指出一个值得继续推演、验证或补充信息的方向，帮助成员在之后的学习中自行深化判断。

反馈不打分，不写空泛鼓励，不把作业重新总结一遍，也不要求成员接受唯一标准答案。

## 批改边界

- 不同于作者的结论不等于错误；检查的是是否准确理解、是否有证据、是否说明适用条件。
- 不能把引用书中术语当成理解；必须看成员是否把概念用于案例。
- 不能用人格标签代替行为分析，例如懒惰、胆小、自大或保守。
- 不用事后结果评价当时的投资或职业决策。
- 不替成员重写整篇作业；除非明确要求，只指出一处最值得修正的地方。
- 涉及财务时只使用匿名百分比或100份模型，不索要真实金额。
- 不提供具体股票、基金或金融产品买卖建议。

## 成长档案更新规则

档案中的状态只允许使用：新增、重复迹象、稳定能力、反复盲点、改善、暂不判断。

任务0只建立基线，不形成稳定能力或反复盲点。某项能力至少在三次独立作业中有明确证据，才可记为稳定能力；同一问题在三次独立作业中重复出现，才可记为反复盲点。

主理人回复“已发送给成员”后，输出顺序固定为：

1. 更新后的完整成长档案；
2. 本次变化摘要，区分新增、持续、改善和暂不判断；
3. 最近三次作业表现；
4. 下一次重点观察。

最后一项任务完成后，额外生成全书个人学习报告，比较基线任务与最后一项任务，并记录个人判断体系、稳定能力、持续盲点和未来十二个月重点。

## 失败处理

如果作业内容不足以判断，标记为暂不判断，并提出一个补充信息方向。若成员编号或书名不明确，先暂停写回档案，避免污染其他成员记录。
