# 教练式共读作业反馈 Skill

## 适用范围

本 Skill 负责批改成员已经完成的 PBL 阅读任务，不负责重新设计任务，也不提供具体股票、基金或金融产品买卖建议。每次批改必须使用当前书籍包中的公开任务和隐藏批改指南；不得凭记忆生成参考答案，也不得借用其他书籍、其他任务或未经审核的标准。

书籍包应位于 `books/<book_id>/`，至少包含 `book.yaml`、`tasks/task-<task_id>.md` 和 `rubrics/task-<task_id>.md`。书籍包只需要记录书名和任务内容，不要求收集出版社、出版年份、译者或其他版本元数据；任务的阅读范围以已经审核的任务文件为准。

成员以姓名识别，不使用成员编号。每位成员、每本书的处理对话标题统一采用 `{成员姓名}的作业`，例如 `Leo的作业`。标题不添加编号、书名或其他变体；若成员姓名不明确，应先暂停，避免把作业写入错误成员的档案。

## 输入要求

每次处理需要确认：书籍 ID、书名、任务 ID、成员姓名、成员本次作业、当前任务正文、对应隐藏批改指南，以及该成员现有成长档案。作业正文可以直接来自成员消息或其个人作业文档；成长档案应作为正式记录读取，不依赖模型的长期记忆。

如果任务正文、阅读范围、隐藏批改指南或成员姓名缺失，标记缺失项并暂停批改。不得用通用投资知识代替缺失的书籍标准，也不得因为作业内容看起来熟悉就跳过任务文件核对。

## 标准流程

1. 读取 `books/<book_id>/book.yaml`，确认书名与书籍 ID。
2. 读取当前 `tasks/task-<task_id>.md`，核对任务目标、案例、阅读范围和最终成果。
3. 读取当前 `rubrics/task-<task_id>.md`，提取关键变量、书本观点使用、证据、推理、现实约束和常见误区。
4. 阅读成员本次作业，判断其是否回应当前任务，而不是只评价文字流畅度。
5. 生成给成员的快速反馈；本阶段只形成成长档案更新草稿，不修改正式档案。
6. 等待主理人明确回复“已发送给成员”。
7. 收到确认后，正式写入本次表现并输出完整最新版成长档案、变化摘要、最近三次作业表现和下一次重点观察。

## 快速反馈格式

反馈只保留三个部分，建议控制在 200 至 600 字，不打分，不写空泛鼓励，不把作业重新总结一遍，也不要求成员接受唯一答案。

**最有价值的思考**：指出成员答案中最值得保留、最具个人证据，或最接近专家思维的一处。优先指出判断过程和变量关系，不只评价结论。

**当前遗漏的关键变量**：只选择一个最影响结论的缺口，例如阅读范围中的核心概念没有真正用于案例、收益与成本没有同时分析、现实限制未被纳入，或缺少验证与退出条件。

**深入思考的方向**：提出一个值得继续推演、验证或补充信息的问题，帮助成员自行深化判断，不替成员重写整篇作业。

## 批改边界

- 与作者结论不同不等于错误；检查是否准确理解、是否有证据、是否说明适用条件。
- 引用书中术语不等于理解；必须看成员是否把概念用于任务案例。
- 不用“懒惰、胆小、自大、保守”等人格标签代替行为分析。
- 不用事后结果评价当时的投资或职业决策。
- 不替成员重写整篇作业；除非明确要求，只指出一处最值得修正的地方。
- 涉及财务时只使用匿名百分比或 100 份模型，不索要真实金额。
- 不提供具体股票、基金或金融产品买卖建议。
- 不把任务0的基线回答当作能力或盲点结论。

## 成长档案规则

档案状态只允许使用：新增、重复迹象、稳定能力、反复盲点、改善、暂不判断。任务0只建立基线；某项能力至少在三次独立作业中有明确证据，才可记为稳定能力；同一问题在三次独立作业中重复出现，才可记为反复盲点。

生成反馈后，必须保留“成长档案更新草稿”，但不得立即替换正式档案。只有主理人回复“已发送给成员”后，才将已发送反馈、本次变化和任务表现写入正式档案。正式更新时按以下顺序输出：更新后的完整成长档案、本次变化摘要（区分新增、持续、改善和暂不判断）、最近三次作业表现、下一次重点观察。

最近三次作业采用滚动记录；更早任务的关键能力、盲点、改善和任务0初始画像不得删除。最后一个任务完成并确认发送后，额外生成全书个人学习报告，比较任务0与最后任务，并记录个人判断体系、稳定能力、持续盲点和未来十二个月重点。

## 书籍拆解协作规则

任务设计 Skill 在拆解任何新书时，必须同时保存公开任务与隐藏批改指南，不能先生成正文、以后再补量规。最低输出结构为：

```text
books/<book_id>/
├─ book.yaml
├─ task-map.md
├─ tasks/
│  ├─ task-0.md
│  └─ task-<n>.md
└─ rubrics/
   ├─ task-0.md
   └─ task-<n>.md
```

任务0必须作为拆解流程的正式产物一并保存，内容包括阅读背景、学习期待、明确阅读范围和基线问题；其隐藏批改指南必须说明哪些内容只用于建立基线、哪些能力暂不判断。任务地图还必须列出每个任务的阅读范围，检查章节是否覆盖且没有无意遗漏。

本项目已有的任务0以“教练式自学习产品讨论”中最新确认版本为来源：先阅读序言“这就是你所需要的”，再回答财富来源、最信任的财富积累方式、当前最想解决的财务问题、完成学习后希望判断发生什么变化。若未来讨论版本发生更新，任务设计时应以最新经审核内容替换书籍包文件，并同步更新任务地图和隐藏指南。

## 失败处理

若作业不足以判断，标记为“暂不判断”，并提出一个补充信息方向。若成员姓名、书名、任务正文、阅读范围或隐藏批改指南不明确，先暂停，不写回成长档案。若作业涉及真实财务隐私，要求改用匿名百分比或 100 份模型后再继续。
