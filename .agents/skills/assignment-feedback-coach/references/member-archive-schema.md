# 成员档案结构

## 目录结构

```text
members/
├─ member-index.md
└─ <member_id>/
   ├─ profile.md
   └─ books/
      └─ <book_id>/
         └─ growth-profile.md
```

`member-index.md` 是身份与进度入口，只记录稳定成员标识、当前昵称、历史昵称、在读或已完成书籍、最近任务和最近更新时间。`profile.md` 是个人跨书总览，只记录跨书阅读进度、由多本书证据支持的长期观察，以及各单书档案链接。`growth-profile.md` 是单书正式档案，保存任务0初始画像、已发送反馈、本次变化、最近三次作业、后续观察和全书学习报告。

## 身份规则

使用不可变的 `member_id` 作为目录名，使用可变的昵称作为对话标题和成员可见称呼。默认 `member_id` 由规范化昵称和三位顺序号组成，例如 `leo-a01`；一旦分配不得因昵称变化而修改。分配前在成员索引的当前昵称和历史昵称中进行不区分大小写的精确查找；出现多个候选、疑似重名或无法确认的旧目录时，暂停并请主理人确认。

新成员在反馈尚未确认发送前，只保留档案更新草稿，不创建正式目录。收到“已发送给成员”后，先在索引中分配身份，再创建跨书总览和单书档案。对已有成员，正式更新顺序为单书档案、跨书总览、成员索引；任一步失败时报告不一致，不得假装全部写入成功。

## 成员索引格式

成员索引使用 Markdown 表格，字段固定为：`member_id`、`当前昵称`、`历史昵称`、`书籍`、`最近任务`、`状态`、`最近更新`。历史昵称为空时写 `—`；书籍记录使用书籍 ID；状态只使用“在读”或“已完成”。同一成员只占一行，同一昵称不得映射到多个活跃成员，除非主理人提供额外区分信息。

## 跨书总览格式

跨书总览至少包含成员身份、书籍进度、跨书观察和单书档案链接。跨书观察不得从一次作业直接推断；只有某种表现已在至少两本书中出现，才记录为“跨书重复迹象”，至少三本书中出现才记录为“跨书稳定能力”或“跨书反复盲点”。单书内部的三次证据规则继续适用，两类证据不得混用。

## 单书档案格式

单书档案按顺序包含：当前进度、完整成长档案、已发送反馈、本次变化摘要、最近三次作业表现、下一次重点观察，以及结书后生成的全书个人学习报告。任务0初始画像永久保留；最近三次作业只滚动作业摘要，不删除更早任务已经形成的能力、盲点、改善或关键基线。

## 迁移规则

发现 `members/<昵称>/` 形式的旧目录时，先读取其中档案确认成员昵称和书籍，再检查成员索引是否已有映射。只有昵称唯一、内容一致且目标 `member_id` 目录不存在时，才可迁移；迁移后更新档案内链接和成员索引。若昵称重复、目标目录已存在或内容身份不一致，停止迁移并请主理人决定，不得覆盖、合并或删除任何档案。
